SuperNOVAS release checklist

Preparation

Version update

  • In novas.h:
    • change doxygen @version in top description (also in novas.c).
    • change version constants (below imports), including release string!
  • README.md: version statement near the top of the document.
  • In Doxyfile: (API version w/o patchlevel)
  • CHANGELOG.md: [Unreleased] to [1.x.x] -- YYYY-MM-dd

Check CHANGELOG

  • Check for completeness
  • Check that changes are listed in the most appropriate section
  • Check for language, spelling, typos

Check build

$ make all

then

$ make distclean

Check build on LynxOS PowerPCs also!

Commit and push

Commit message is version v1.x.x, e.g. v1.1.0 or v1.1.0-rc2.

Release

GitHub

  • Create a new release. Copy/paste section from CHANGELOG.md as the release description.
  • Close Milestone (and create next Milestone(s) as appropriate).
  • Unpin any closed issues.
  • In Discussions, unpin prior release(s), also from Announcements, and pin current release. Close discussions relating to prior releases.

Back to devel

  • Bump version in novas.h for next releaze, and set release string to -devel. Commit and push to main.

Social

  • Post release blob to Bluesky, with site link and #astronomy tag.
  • Also to LinkedIn, and Facebook (Astronomer's group). Use #astronomy tag as appropriate.
  • Post on CfA Slack.
